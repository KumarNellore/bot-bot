bot-bot by Imaginea
Selenium like automation framework for native android applications.

Components

bot-bot have following three different components in it:

  • Server - the server is used to record user actions through Recorder. It also allows the users to export their test-cases so that they can be run with the Runner.
  • Recorder - Recorder is used to record user actions at the android app side and then send them to the Server.
  • Runner - This is the main part of bot-bot. The runner allows the user to run their recorded test-cases on the actual android application as a suite. It also generated emailable html reports for your test execution.

Server

Introduction:

Bot-bot server is used to record, store and modify the user actions taken on the android app at the user end/system.
It is deployed on a web-servlet supported webserver and currently uses a h2 database to persist the data.

It have the following functionalities:

  • View recorded sessions
  • View recorded entries for a particular session.
  • Persist recorded sesions.
  • Allow users to edit recorded sessions
  • Allow users to export the recorded steps into a "csv" file. This file can then be used with bot-bot runner for you test-suite exeution.

Recorder

Introduction:

Recorder is used to record user actions on the android app under test and send these actions to the bot-bot server. The recording can be done through simulator or using an actual phone.

Recorder will record the basic commands for your user-actions. These steps can then be modified to make an actual test-scenarios by inserting extra commands like asserts,verify,waitfor etc.
Recorder creates a new session with the server whenever the app is launched. Once a session is created further recording commands are sent to the said session.
Recorder records most of the user action done on the android app. Some of the actions like a back key on the keyboard cannot be recorded, as it may be over a softkeyboard or a text box.
If you need to add such scenarios, you can modify the recorded scripts to simulate required key-press.

Runner

Introduction :

Runner is the component that actually runs your recorded scripts onto your application as a Test-suite.

bot-bot is based on keyword driven approach and uses "csv" to define its test-cases. The server & recorder records the test-cases as required by the runner. Runner supports 2 types of frameworks NativeDriver & Robotium. User can choose accordingly from any of the said frameworks by mentioneing the same in the properties configuration.

Runner internally uses following open-source tools for test execution and reporting:

  1. Robotium
  2. Selenium Nativedriver
  3. TestNG
To run your testcases as a test-suite, just place your test-cases(which are in "csv" format) in folders(Used as module) or sub-folders(sub-modules) under a single folder for your app under resources folder of the runner.
Also mention your suite folder name and app details inside the default.properties file present in the root of the runner.
When you run the runner , it will automatically identify the files under folders and sub-folders and accordingly execute them. At the end of the test execution runner will generate html report with your fodlers/modules & sub-folders/submodules defined in the same hierarchy as defined by you.
