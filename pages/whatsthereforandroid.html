Whats already there for android

There are only few opensource tools that are available in the market that can be used for android automation.
Following are few of them:

1. Selenium Native Driver
	Pros:
	- Easy to integrate.
	- Uses java to write the cases.
	- Written over selenium(the most widely used opensource automation tool)>
	Cons:
	- Test cases needs to be written using the java code.
	- Recording option not available
	- Requires application source code for automating and integrating.

2. Sikuli
	Pros:
	- Image based automation tool.
	- Images/icons/ screens can be verified.
	- No integration with the code is required.
	Cons:
	- As it is an image based test.Managing images for a huge set of test-cases is difficult.
	- Need to write java code if we need to generate a html report for the execution.

3. FoneMonkey for android
	Pros:
	- Gives recording option.
	- Testcases can be converted to Junit cases.
	Cons:
	- Product is not still stable.
	- Need integration with the application source code.

4. Robotium
	Pros:
	- Do not require integration with the source code.
	- Can run on any given application apk file.
	Cons:
	- Slow execution.
	- Noe report generation
